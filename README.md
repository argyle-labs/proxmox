# proxmox

A standalone [orca](https://github.com/argyle-labs/orca) plugin for
**Proxmox VE**. It adapts a Proxmox cluster's API into orca: an endpoint
registry, node/guest inspection and lifecycle actions, cluster-quorum
reporting, and — across the cdylib ABI — a `cluster_roster` provider and a
`topology` collector that restore fleet grouping and parent-host topology with
no static link into orca.

The typed PVE client is code-generated by `progenitor` from a vendored upstream
spec (`specs/proxmox.openapi.json`, derived from a node's `apidoc.js`);
`build.rs` runs `plugin_toolkit_build::openapi::generate_all` over it.

## How it loads

This crate builds as a `cdylib` (the artifact orca's `plugin-loader` `dlopen`s
at runtime) and as an `rlib` (so the in-crate test harness and a checked-out
orca workspace can use it directly). Its **only** orca dependency is
[`plugin-toolkit`](https://github.com/argyle-labs/orca) — every orca primitive
(`#[orca_tool]`, `endpoint_resource!`, dispatch, contract, http, runtime, …)
resolves through the toolkit and its prelude. On a successful load, orca's
loader registers the plugin's tools plus its two domain backends into the
matching `contract` registries.

## Tool surface

| Tool | Purpose |
|------|---------|
| `proxmox.list` / `detail` / `create` / `update` / `delete` | Endpoint registry — register a Proxmox API target (host, token); CRUD the stored rows. |
| `proxmox.nodes` | List cluster nodes for a registered endpoint. |
| `proxmox.node_detail` | List VMs + containers on one node. |
| `proxmox.action` (admin) | Lifecycle action against a guest (start/stop/etc.). |
| `proxmox.host_logs` | Pull host logs for a node. |
| `proxmox.cluster_status` | Cluster name, quorum, and node membership for one endpoint. |
| `proxmox.cluster_list` | Cluster resources (VMs + LXC) across the cluster. |

Two further contributions reach orca over the ABI seam (`src/abi_export.rs`)
rather than as operator-facing tools:

- **`cluster_roster`** (`src/cluster_roster_impl.rs`) — supplies fleet cluster
  grouping (`list_clusters`).
- **`topology`** (`src/topology.rs`) — emits parent-host claims
  (`collect_claims`) for every host with a registered Proxmox endpoint, so
  Proxmox-hosted guests surface in orca's systems graph.

## Build & register

```sh
cargo build --release          # produces target/release/libproxmox.{dylib,so}
orca plugin add /abs/path/to/orca-plugin.toml   # register with orca
orca plugin list
```

orca `dlopen`s the built cdylib from its plugin directory; "deploying" this
plugin means building the cdylib and registering it — there is no container
image or service of its own. The committed `[patch]` in `.cargo/config.toml`
points `plugin-toolkit` at a local in-tree orca for development; an external
checkout resolves it from the `argyle-labs/orca` git branch.

## Refresh the PVE spec

Overwrite `specs/apidoc.js` + `specs/proxmox.openapi.json` from a cluster node
and rebuild — `build.rs` regenerates the typed client.
